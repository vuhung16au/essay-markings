# UI/UX Interaction Design Policy

## Goals

- Make the app easy to use for learners.
- Keep the result report clear, supportive, and actionable.

## Rules

- Prefer clarity over decoration.
- Show the most important information first.
- Make score breakdowns easy to scan.
- Keep feedback readable in both dark and light themes.
- Make export actions obvious and reliable.
- Ensure user-facing copy sounds supportive, not mechanical.

## Result design guidance

- Explanations should help revision, not just justify scoring.
- Deduction reasons should be concise and understandable.
- Strengths and improvements should be visually distinct.
